# Мониторинг можно разделить на следующие виды 
* **Аггрегатор событий**
* **Графическое представление**
* **Нотификация**

## Варианты использования
- трешхолды на метрики для раннего реагирования и предотвращения аварийных ситуаций

Пример: собираем две метрики. Выделенное количество памяти  для спарк задачи me, и фактическое потребление mf. Устанавливаем трешхолд на 80%≥ mf/me*100. При достижении трешхолда принемаем меры для недпущения аварии.

- Ведение матрицы проблем. По оси Y ведем датасеты , по оси X ведем нарушение трешхолдов, пересечение заполняется количеством нарушений за указанный период.
